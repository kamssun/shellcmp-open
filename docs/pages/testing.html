单元测试

命令说明
./gradlew :a-core:allTestsCore 模块测试
./gradlew :a-platform:allTestsPlatform 模块测试
./gradlew :a-shared:allTestsShared 模块测试
./gradlew :a-shared:desktopTestShared Desktop 测试（最快，无需设备）
./gradlew :a-shared:desktopTest --tests "*.ChatDaoStressTest"Chat 压力测试
./gradlew :a-shared:desktopTest --tests "*.ChatDaoWindowTest"Chat 窗口查询测试

覆盖率门禁

硬性要求：≥80%（LINE，JaCoCo 引擎）

低于阈值构建直接失败，不允许"先上线再补测试"。覆盖率由 Koin 完整 DI 环境保证可信度。

命令说明
./gradlew koverHtmlReport全模块聚合覆盖率报告（HTML）
./gradlew koverLog全模块覆盖率摘要（控制台）
./gradlew koverVerify全模块覆盖率验证（≥80%，构建失败时用）

Preview 截图对比

通过 generateComposePreviewRobolectricTests 自动扫描所有 @Preview 函数生成截图测试，无需手写测试类。新增 Preview 即自动纳入覆盖，private Preview 会被排除。

关键优势

Preview 基于 Koin 启动完整的依赖注入环境（DI + 导航 + i18n + 主题），不是假数据的静态预览，而是接近真实运行状态的组件渲染，截图对比的可信度远高于普通 Preview。

命令说明
./gradlew :androidApp:recordRoborazziDebug录制基准截图（首次或变更后执行）
./gradlew :androidApp:verifyRoborazziDebug验证截图差异（CI 上执行）

VF 截图回归

录制 → ADB 回放 → SSIM 截图对比。详见 VF 全自动验证文档。

命令说明
./gradlew verifyVf全量 VF 验证
./gradlew verifyVf -Pvf.dir=<path>指定目录验证
./gradlew cleanVerification清除验证产物
/update-vfAI 操控手机 + TimeTravel 浮窗录制

录制方式

手动录制
App 内 TT 浮窗 → 「录制开始」→ 操作 → 「录制结束」→ 保存 zip
AI 自动录制
/update-vf 通过 adb-mcp 操作手机模拟人工交互，一条命令搞定

E2E 测试（Maestro）

支持 Android + iOS 真机全流程测试，验证真实设备上的完整用户路径。

# iOS
maestro test .maestro/flows/ -e APP_ID=com.example.archshowcase.ArchShowcase

# Android
maestro test .maestro/flows/ -e APP_ID=com.example.archshowcase

# 多设备同时连接时需指定 udid
maestro test .maestro/flows/ --udid <device-udid>

性能基准测试

# 生成 Baseline Profile（需连接设备 API 28+）
./gradlew :androidApp:generateBaselineProfile

# 启动基准测试
./gradlew :macrobenchmark:connectedBenchmarkAndroidTest
