KSP 代码生成

标两个注解
其余全自动

四个注解消除大量重复代码。新增一个带回溯 + 全自动验证的页面 = 写业务代码 + 标两个注解，胶水代码、路由注册、验证接入全部自动生成。新人看到注解就知道这个模块具备哪些能力。

@Replayable @RouteRegistry @VfResolvable @CustomState

概述

实际感受

写新页面时完全不需要关心"回溯怎么接入"、"VF 怎么注册"、"路由怎么序列化"这些基础设施问题，标上注解就能用，心智负担极低。

注解作用生成内容
@Replayable接入交互回溯体系ExportStrategy + FactoryHelpers
@RouteRegistry路由注册路由序列化 + 注册代码
@VfResolvable接入 VF 验证体系VF 解析器 + 注册入口
@CustomState字段订阅 + 内存快照SelectableState + MemoryTrackable

四个注解一览

接入成本
新增带回溯 + 全自动验证的页面 = 写业务代码 + 标 2 个注解（@Replayable + @VfResolvable），其余全部自动生成。
可读性
注解即文档。看到 @Replayable 就知道这个模块支持时间旅行，看到 @VfResolvable 就知道有全自动验证覆盖，不需要翻文档。
@Repeatable 支持
@Replayable 支持 @Repeatable，同一 Record 可标注多个注解，为不同 Store 各自独立生成一套代码。
生成文件位置
a-shared/build/generated/ksp/metadata/commonMain/，每个注解实例独立生成文件，不互相污染。

@Replayable — 接入交互回溯

标注在 HistoryRecord 上，KSP 自动生成 ExportStrategy 并注册到 StoreExportRegistry。

@Replayable  // storeName 为空时从父类推导
@Serializable
data class MyHistoryRecord(
    val type: MyHistoryType,
    val timestamp: Long
) : JvmSerializable {
    fun applyToState(prevState: State): State = ...
    fun toIntent(): Any = ...
}

生成产物：

文件内容
{Store}ExportStrategy.ktSimpleExportStrategy 实现，包含 createInitialState、processRecord
GeneratedExportStrategies.kt统一注册入口，registerGeneratedExportStrategies() 一次注册所有策略
@Repeatable 用法

同一 Record 可多次标注，为不同 Store 各自生成代码：

@Replayable(stateClass = NavigationStore.State::class)
@Replayable(stateClass = NavigationStore.State::class, storeName = "DemoNavigationStore")
data class NavHistoryRecord(...)

@RouteRegistry — 路由注册

新增页面只需一行注解声明，路由注册、序列化代码全部自动生成，无需手写。

@RouteRegistry
@Serializable
data class ImageDetailRoute(val imageId: String) : AppRoute

// KSP 自动生成：
// - 路由序列化适配器
// - RouteRegistry 注册代码
// - Decompose ChildFactory 分支
效果

新增一个路由 = 定义 data class + 加一行 @RouteRegistry，不需要手写注册逻辑、序列化、ChildFactory 分支。

@VfResolvable — 接入 VF 验证

标注在业务模块上，自动生成 VF 解析器并注册到验证体系，新模块无需手动配置。

@VfResolvable
class ImageDemoComponent(...) : ImageDemoComponent {
    // 业务逻辑...
}

// KSP 自动生成：
// - VfResolver 实现
// - GeneratedVfResolvers.kt 统一注册入口
与 @Replayable 联动

@VfResolvable 依赖 @Replayable 提供的状态回溯能力。两个注解组合，新页面同时获得：时间旅行调试 + 全自动 VF 回归验证。

@CustomState — 字段订阅 + 内存快照

复合注解，等价于同时标注 @SelectableState + @MemoryTrackable，一次标注同时生成两套代码。

@SelectableState（字段订阅）
生成细粒度的字段级订阅扩展，让 UI 只订阅它真正关心的字段，避免无关字段变化触发重组。
@MemoryTrackable（内存快照）
生成 toMemorySnapshot()，递归统计 Collection/Map 的 .size，用于 GC 压力检测中的 Store 集合大小报告。
@CustomState  // = @SelectableState + @MemoryTrackable
data class ChatRoomState(
    val messages: List<Message> = emptyList(),
    val isLoading: Boolean = false,
    override val history: AppendOnlyHistory<ChatHistoryRecord> = AppendOnlyHistory()
) : ReplayableState<ChatHistoryRecord> { ... }

// 生成后可用：
state.selectMessages()        // 仅订阅 messages 字段
state.toMemorySnapshot()      // 输出到 GC 压力报告
