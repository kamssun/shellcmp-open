Four annotations eliminate massive boilerplate. Adding a new page with state replay + fully automated validation = write business code + add two annotations — glue code, route registration, and validation wiring are all auto-generated. New developers can see at a glance what capabilities a module has just by looking at its annotations.

@Replayable — Integrate with State Replay

Annotate on HistoryRecord — KSP auto-generates ExportStrategy and registers it with StoreExportRegistry.

@Replayable  // storeName inferred from parent class when empty
@Serializable
data class MyHistoryRecord(
    val type: MyHistoryType,
    val timestamp: Long
) : JvmSerializable {
    fun applyToState(prevState: State): State = ...
    fun toIntent(): Any = ...
}

@Repeatable Usage

The same Record can carry multiple annotations, generating independent code for each Store:

@Replayable(stateClass = NavigationStore.State::class)
@Replayable(stateClass = NavigationStore.State::class, storeName = "DemoNavigationStore")
data class NavHistoryRecord(...)

@RouteRegistry — Route Registration

Adding a new page requires only a single annotation — route registration and serialization code are fully auto-generated with no manual wiring.

@RouteRegistry
@Serializable
data class ImageDetailRoute(val imageId: String) : AppRoute

// KSP auto-generates:
// - Route serialization adapter
// - RouteRegistry registration code
// - Decompose ChildFactory branch
Result

Adding a new route = define a data class + add one @RouteRegistry annotation — no need to hand-write registration logic, serialization, or ChildFactory branches.

@VfResolvable — Integrate with VF Validation

Annotate on business modules — auto-generates VF resolvers and registers them with the validation system. New modules require no manual configuration.

@VfResolvable
class ImageDemoComponent(...) : ImageDemoComponent {
    // Business logic...
}

// KSP auto-generates:
// - VfResolver implementation
// - GeneratedVfResolvers.kt unified registration entry
Works with @Replayable

@VfResolvable relies on the state replay capability provided by @Replayable. Combined, a new page gains both: time-travel debugging + fully automated VF regression validation.

@CustomState — Field Subscription + Memory Snapshot

A composite annotation equivalent to applying @SelectableState + @MemoryTrackable together — one annotation generates two sets of code.

@SelectableState (Field Subscription)
Generates fine-grained field-level subscription extensions so the UI only subscribes to the fields it actually cares about, avoiding recomposition triggered by unrelated field changes.
@MemoryTrackable (Memory Snapshot)
Generates toMemorySnapshot() which recursively counts Collection/Map .size values, used for Store collection size reporting in GC pressure detection.
@CustomState  // = @SelectableState + @MemoryTrackable
data class ChatRoomState(
    val messages: List<Message> = emptyList(),
    val isLoading: Boolean = false,
    override val history: AppendOnlyHistory<ChatHistoryRecord> = AppendOnlyHistory()
) : ReplayableState<ChatHistoryRecord> { ... }

// Available after generation:
state.selectMessages()        // Subscribe to messages field only
state.toMemorySnapshot()      // Output to GC pressure report
