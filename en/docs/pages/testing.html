Multi-Layer Testing System

Five-Layer Defense
Fully Automated

Five layers of defense from fast to slow, from fine-grained to coarse. Unit tests ensure function correctness, coverage ensures no blind spots, Preview screenshots ensure component visuals, VF ensures complete user paths, E2E ensures real devices in real environments — no manual regression needed.

Coverage ≥80% Roborazzi VF Fully Auto Maestro E2E

Coverage Gate

Hard requirement: ≥80% (LINE, JaCoCo engine)

Build fails below the threshold — no shipping without tests. Coverage credibility is ensured by the full Koin DI environment.

CommandDescription
./gradlew koverHtmlReportAggregated coverage report for all modules (HTML)
./gradlew koverLogCoverage summary for all modules (console)
./gradlew koverVerifyCoverage verification for all modules (≥80%, use when build fails)

Preview Screenshot Comparison

Uses generateComposePreviewRobolectricTests to auto-scan all @Preview functions and generate screenshot tests — no handwritten test classes needed. New Previews are automatically included in coverage; private Previews are excluded.

Key Advantage

Previews boot the full Koin dependency injection environment (DI + navigation + i18n + theming) — not static previews with fake data, but component rendering close to real runtime state, making screenshot comparison far more reliable than ordinary Previews.

CommandDescription
./gradlew :androidApp:recordRoborazziDebugRecord baseline screenshots (run on first use or after changes)
./gradlew :androidApp:verifyRoborazziDebugVerify screenshot differences (run on CI)

E2E Tests (Maestro)

Supports full-flow testing on Android + iOS real devices, verifying complete user paths on actual hardware.

# iOS
maestro test .maestro/flows/ -e APP_ID=com.example.archshowcase.ArchShowcase

# Android
maestro test .maestro/flows/ -e APP_ID=com.example.archshowcase

# Specify udid when multiple devices are connected
maestro test .maestro/flows/ --udid <device-udid>
