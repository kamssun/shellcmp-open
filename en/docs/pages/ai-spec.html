OpenSpec — AI-Driven Complex Feature Development

Large cross-module features go through the OpenSpec structured workflow, ensuring design intent is preserved throughout implementation.

01

AI-Generated Proposal

Describe the feature goal, and AI automatically analyzes the impact scope and generates a technical proposal.

02

Technical Design

Detailed design of module interfaces, data flows, and key decision points, with optional manual review and modifications.

03

Task Breakdown

Break the technical design into independently executable subtasks with clear dependencies and acceptance criteria.

04

Step-by-step Implementation per Spec

Each implementation step follows the spec, ensuring the final code is consistent with the design intent.

Documentation Principles

Minimalism
If it can be said in one sentence, don't use two. No redundant descriptions, repeated information, or decorative text. Choose the right format: a single rule gets one sentence, multiple mappings use tables, indexes over full text.
Progressive Disclosure
Information is layered on demand, avoiding information overload. L0 contains only indexes and mental models, details sink to L1-L3, each layer focuses on a single domain.
Refinement Criteria
MEMORY.md is a scratchpad. Patterns or lessons learned that have been validated across multiple sessions and are no longer changing get refined into yaml/docs and removed from the scratchpad.
Adding New Specs
First determine the layer (L0-L3) for a new spec, then choose the placement. Each L1/L2 file focuses on a single domain with no cross-references.
