Fully Automatic Event Tracking

Zero Handwritten Code
Full Collection of Four Event Types

Fully automatic collection of four event types: Page, Interaction, Exposure, and Foreground/Background. Adding a tracked list page = ExposureLazyColumn + appClickable — no tracking code needed. Event names, business parameters, and sanitization are all handled automatically.

KSP Auto-Mapping Three-Layer Sanitization Exposure Tracking Zero Configuration

Architecture

User Action → InteractionContext Tagging → Store.accept(Intent) ↓ TrackingExecutor Intercept ↓ KSP EventMapper (Type Matching) ↓ RuntimeSanitizer Sanitization ↓ AnalyticsCollector Reporting

Four event types fully automated: Page (routing), Intent (interaction), Exposure (impression), and Foreground/Background.

Intent Collection Rules

Core Principle

All user interaction entry points must go through the appClickable family. Direct use of Compose native interaction APIs is prohibited.

ScenarioUseReplaces
ClickModifier.appClickable(component, onClick).clickable
Long press / Double tapModifier.appCombinedClickable(...).combinedClickable
Keyboard IMEappKeyboardActions(component, onSend, ...)KeyboardActions
SwitchAppSwitch (built-in tracking)—
Filter chipAppFilterChip (built-in tracking)—
ButtonAppButton / AppTextButton / AppIconButton—

appClickable Parameters

Modifier.appClickable(
    component = "ChatInputBar",        // Component identifier for logging
    gestureType = GestureType.TAP,     // TAP / LONG_PRESS / SWITCH / CHIP / IME_ACTION
    enabled = true,
    interactionSource = interactionSource, // Optional, pass when press animation is needed
    role = Role.Switch,                // Optional, semantic role
    onClick = { ... },
)
Interactions That Don't Need Tracking

Pure UI effects (scrolling, dragging, animation triggers) do not fire Store Intents, do not need appClickable, and can continue using native .clickable.

Exposure Impression Collection

ExposureLazyColumn automatically tracks all items with key + contentType, including KSP auto-extracted business parameters, with zero manual code.

ExposureLazyColumn(
    listId = "image_list",
    state = listState,
) {
    items(
        items = images,
        key = { it.id },               // Used as exposureKey
        contentType = { "image_card" }  // Used as componentType
    ) { image ->
        ImageCard(image)               // No Modifier needed
    }
}

Auto-Tracking Rules

ConditionBehavior
Has key + has contentTypeAutomatically injects trackExposure, exposureKey = {contentType}_{key}, params = KSP extracted
Missing key or contentTypeSkipped, not tracked

Exposure Trigger Conditions

Visible Area >= 50%
Calculates visible ratio based on LazyList's visibleItems, accurately determining whether the item truly entered the viewport.
Dwell Time >= 500ms
Prevents false triggers during fast scrolling. Only counts as an exposure when the user actually stops to view.
Per-Page Deduplication
Within the same page lifecycle, each key is reported only once, preventing repeated triggers from scrolling.
Auto Business Parameter Extraction
KSP scans data classes and auto-generates toExposureParams(), automatically attaching business fields on exposure.

KSP Auto-Generation

EventMapper (Intent → TrackingEvent)

  • Automatically scans all sealed interface Intent in Store interfaces
  • Generates {StoreName}EventMapper.kt + EventMapperRegistry.kt
  • Registry matches by Intent type (when (intent)), no dependency on storeName strings
  • Naming: CamelCase → snake_case (e.g., SendText → send_text)
  • New Intents are reported with zero configuration

ExposureParamsProcessor

Annotation-free full scan of all data classes under com.example.archshowcase.*, generating toExposureParams() extensions that reuse ParamsCodeGenerator sanitization rules.
