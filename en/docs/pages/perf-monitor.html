PERF Log ↔ Perfetto Trace Mapping

Combined with Perfetto traces captured by btrace, PERF log aggregate data can be mapped to method-level slices.

Key Point: Compose Frame Rendering Model

Compose bypasses the View system's onMeasure/onLayout, so the layout field is usually 0. measureAndLayout appears in two locations (inside ANIMATION callback + inside TRAVERSAL callback) — when analyzing traces, you must distinguish them by their parent chain.

Example: Frame Breakdown Log vs Trace

Log: delay 21ms | anim(recompose→apply) 69ms | draw(m&l→canvas) 18ms | gpu 1ms

Trace mapping:
doFrame 88ms
├─ CallbackRecord.run (ANIMATION)
│  └─ runRecomposeAndApplyChanges 69ms
│     ├─ performRecompose 55ms       ← recomposition: compute UI tree diff
│     └─ applyChanges 11ms          ← commit diff to SlotTable
└─ TraversalRunnable (TRAVERSAL) ≈ draw 18ms
   └─ performDraw
      ├─ measureAndLayout 13ms       ← Compose measure + layout
      └─ LayoutNode.draw ~5ms*       ← Canvas recording
                                       *btrace collapses to 0ms
+ RenderThread Running 1.4ms         ≈ gpu 1ms
btrace draw Duration Correction

btrace collapses nested slices inside TraversalRunnable to the same duration, causing LayoutNode.draw to show 0ms. In Perfetto, compare the right edge of the slice with the right edge of thread_state Running — the difference is the swallowed duration.

Debugging Workflow

1. Check JANK logs to determine jank type ├─ Choreographer.doFrame slow → rendering issue → check FrameMetrics breakdown │ ├─ anim high → heavy recompose → check recomposition scope │ ├─ draw high → measureAndLayout or Canvas recording heavy → simplify layout │ └─ delay high → pipeline stall → previous frame too slow, causing queue ├─ Other Message slow → main thread has expensive task → move to background thread ├─ gc blocking → GC pause → reduce object allocation ├─ cpu% low → main thread blocked → btrace stack capture to locate lock/IO └─ No msg + cpu% low → thread scheduling delay / GC concurrent phase 2. btrace captures method-level durations; in Perfetto, Ctrl+F search "PERF:JANK" to jump directly to the jank frame 3. After fix, compare JANK logs to verify
