Core Concepts

More Than Screenshot Comparison

VF records complete user operation paths + server-side data snapshots. Not only UI styling issues, but logic bugs on critical paths (data calculation errors, abnormal state transitions, wrong conditional branches) will be caught as long as they are ultimately reflected in the interface. When all critical paths are covered by recordings, it is theoretically equivalent to a full regression test suite with zero hand-written assertions.

Why It Is an Order of Magnitude Faster Than Traditional E2E

Traditional E2E
Find widget -> tap button -> wait for animation -> find next widget. Every step incurs lookup and wait overhead, constrained by rendering speed.
VF Approach
Directly injects operation commands into the business layer for replay -- no need to simulate touches, traverse UI elements, or wait for animations. State restoration and command replay are pure in-memory operations; rendering is only involved when taking the final screenshot.

Validation Workflow

01

Change Analysis

AI automatically analyzes code changes, identifies affected test cases, and flags new features not yet covered.

02

Path Inference

Reads business code, automatically enumerates all interaction paths, and generates operation plans -- no manual test step definitions needed.

03

Auto Recording

AI controls a real device to simulate user operations and complete recording, with no manual intervention needed. The recording result contains operation sequences + network snapshots.

04

Screenshot Validation

A custom-built SSIM image similarity engine compares screenshots and outputs difference heatmaps, automatically determining whether unexpected changes exist.

05

Server-Side Data Pinning

Automatically captures network requests/responses during recording; replays the recorded data directly during validation, completely eliminating interference from backend data changes and network environment differences.

Recording Methods

🤖 AI Auto Recording
Done with a single command. AI automatically controls the device to complete recording, covering most common scenarios.
/update-vf
✋ Manual Recording
In the app, tap "Start Recording" on the TimeTravel floating widget -> perform operations -> "Stop Recording" -> save zip. Use when dealing with complex interactions or when AI results are unsatisfactory. Record once, and all subsequent validations are fully automated.

Output Artifacts

Validation results are output to <vf_dir>/output/, containing:

FileContent
verify_a.png / verify_b.pngBefore and after screenshots for comparison
result_ssim_start.json / result_ssim_end.jsonSSIM similarity scores (0-1, higher means more similar)
diff_heatmap_start.png / diff_heatmap_end.pngDifference heatmaps highlighting areas where changes occurred
report.mdFull validation report with difference analysis and recommendations
