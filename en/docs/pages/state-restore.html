Interaction Replay
The Foundation of Everything

A strict unidirectional data flow architecture naturally produces a complete operation history. Every user action is recorded, and state changes are fully traceable and replayable — serving as the foundation for VF validation, recording systems, and server-side data persistence.

Quick Start

Integration requires only four steps; the rest is auto-generated by KSP.

01

Define HistoryType — User Action Types

Use a sealed interface to describe all replayable action types, with each subclass corresponding to a specific user action.

@Serializable
sealed interface MyHistoryType : JvmSerializable {
    @Serializable
    data class Toggle(val enabled: Boolean) : MyHistoryType
}
02

Define HistoryRecord — With @Replayable Annotation

applyToState() and toIntent() are hand-written; KSP auto-generates FactoryHelpers and ExportStrategy.

@Replayable  // storeName is inferred from parent class when empty
@Serializable
data class MyHistoryRecord(
    val type: MyHistoryType,
    val timestamp: Long
) : JvmSerializable {

    fun applyToState(prevState: State): State = when (type) {
        is MyHistoryType.Toggle -> prevState.copy(
            isEnabled = type.enabled,
            history = prevState.appendHistory(this)
        )
    }

    fun toIntent(): Any = when (type) {
        is MyHistoryType.Toggle -> Intent.Toggle(type.enabled)
    }
}
@Repeatable Support

A single Record can have multiple @Replayable annotations, generating independent code for different Stores.

03

Add history Field to State

data class State(
    val isEnabled: Boolean = false,
    override val history: AppendOnlyHistory<MyHistoryRecord> = AppendOnlyHistory()
) : ReplayableState<MyHistoryRecord> {
    override fun hasValidData() = history.isNotEmpty()
    override fun createInitialState() = State()
}
04

Register Store

// In Component
private val store = registerRestorableStore(
    name = "MyStore",
    factory = { storeFactory.create() }
)

// For parameterized pages (multiple instances), use an instance-specific name to avoid state overlap
private val store = registerRestorableStore(
    name = "$BASE_STORE_NAME:$paramId",
    factory = { storeFactory.create(paramId) }
)
Done!

Store state and history are automatically saved/restored. KSP has generated all the glue code.

Scroll Position Restore

List scroll positions can also be persisted, automatically returning to where the user left off after a restart.

Component Side

private val scrollCoordinator: ScrollUpdateCoordinator by inject()

private val (store, scrollRestoreEvent) = registerScrollRestorableStore(
    name = "MyStore",
    factory = { storeFactory.create() },
    getItemCount = { store.state.items.size },
    isUserScrolling = scrollCoordinator::isUserScrolling
)

override fun updateScrollPosition(index: Int, offset: Int) {
    scrollCoordinator.runWithUserScroll {
        store.accept(Intent.UpdateScroll(index, offset))
    }
}

Content Side

val listState = rememberLazyListState()

ScrollRestoreEffect(listState, component.scrollRestoreEvent)

OBOLaunchedEffect(listState) {
    snapshotFlow { listState.firstVisibleItemIndex to listState.firstVisibleItemScrollOffset }
        .debounce(100)
        .collect { (index, offset) ->
            component.updateScrollPosition(index, offset)
        }
}

LazyColumn(state = listState) { ... }
Note

ScrollUpdateCoordinator marks user-initiated scrolling to prevent conflicts between automatic restoration and manual user actions.

Export / Import

The TimeTravel floating panel provides full state snapshot management capabilities.

📤 Export .tte File
Contains all Store states and histories; can be transferred across devices to reproduce the user's exact context.
📥 Auto-Replay After Import
One-click import to see the user's complete interface at that moment, without guessing the action path.
⏪ Timeline Replay
Drag along the timeline to any historical moment; the current Intent information is displayed at the top during replay.
🖱️ Draggable Floating Panel
The panel supports dragging and is automatically constrained within screen boundaries, without blocking the interaction area.

Export Strategy Registration Chain

AppModule: StoreExportRegistry.setExternalRegistrar(::registerGeneratedExportStrategies) ↓ TimeTravelExporter.initStrategies(): StoreExportRegistry.runExternalRegistrar() ↓ GeneratedExportStrategies: register(ImageDemo/OBODemo/Settings...ExportStrategy)

Generated files are located at a-shared/build/generated/ksp/metadata/commonMain/, containing the ExportStrategy for each Store and a unified registration entry point.

Dev Bookmark

When developing deep navigation links, save the current interface state with one click and automatically restore to that screen after restart, saving time from repeatedly clicking through navigation.

💾 Save Bookmark (Blue)
Exports the current TTE to Downloads/dev_bookmark.tte; automatically detected and restored after restart.
🗑️ Clear Bookmark (Red)
Deletes the file and restores the default startup flow without affecting other configurations.

Startup Restore Flow

App Launch → Detect dev_bookmark.tte ├─ Exists & valid → TteStateExtractor extracts state → RestoreRegistry restores → Navigate directly to target screen ├─ Exists but fails → Delete file → Toast "Bookmark expired" → Normal startup └─ Not found → Normal startup
Persistent Bookmark

The file is preserved after restore (not auto-deleted), supporting repeated restores across multiple restarts. VF validation takes priority over bookmark restore.

Code is located in a-core/trace/bookmark/ (interface + restorer) and androidApp/src/debug/bookmark/ (Android implementation).

How It Works

User Action → Store Updates State → Append Record to history ↓ RestoreRegistry Auto-Saves ↓ Restore on Restart/Import ↓ Replay Records Sorted by timestamp

Core Mechanisms

ComponentResponsibility
registerRestorableStore()Auto register/unregister, lifecycle bound to Component
AppendOnlyHistoryO(1) append, replacing O(N) copy of history + record
RestoreRegistryManages state cache, persists across restarts
ExportStrategySerialization/deserialization, auto-generated by KSP
applyToState()Hand-written member function, determines how a Record is applied to State
toIntent()Hand-written member function, converts a Record back into a replayable Intent
