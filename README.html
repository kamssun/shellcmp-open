核心能力

八大核心系统

每个系统独立可用，协同时形成完整的工程质量飞轮

↗
⏪
MVI 状态回溯 — 一切的基石
严格的单向数据流架构天然产出完整的操作历史。每一次用户操作都被记录，状态变化完全可追溯、可重放。是 VF 验证、录制系统、服务端数据固化的根基——没有这个基石，后面的一切都不可能实现。
交互回溯 状态导出/导入 开发书签 泄漏审计 线上 Bug 复现
↗
🤖
AI 全自动验证（VF）
不只是 UI 截图对比——任何导致界面变化的 Bug 都会被捕获。录制完整用户操作路径 + 服务端数据快照，验证时原样重放并对比截图。比传统 E2E 快一个量级。
变更分析 路径推断 自动录制 SSIM 截图对比 服务端数据固化
↗
⏱️
OBO 调度器 — 不卡顿
队列里永远最多只有一个任务。VSYNC 到达时队列几乎是空的，屏障机制正常生效。同时让每个任务成为独立消息，性能监控能精准定位到"就是这个任务慢了 50ms"。
消息积压消除 三方 SDK 拦截 来源级诊断 三端统一
↗
📊
全链路性能诊断闭环
自研运行时性能监控，不依赖第三方，覆盖三端。启动时间线树形展示每一步耗时；每帧自动拆解为 6 个阶段并与 btrace 火焰图精确对齐；btrace + AI 自动分析 trace 数据并给出优化建议。
启动时间线 卡顿诊断 GC 压力检测 帧拆解 + trace 对齐 btrace + AI 分析 主线程慢任务监控
↗
📍
全自动埋点
四类事件（页面、交互、曝光、前后台）全自动采集。新增带埋点列表页 = ExposureLazyColumn + appClickable，不写任何埋点代码，事件名、业务参数、脱敏全部自动完成。
KSP 自动映射 三层脱敏 曝光追踪
↗
⚙️
KSP 代码生成
四个注解消除大量重复代码。新增一个带回溯 + 全自动验证的页面 = 写业务代码 + 标两个注解，其余全部自动生成。新人看到注解就知道这个模块具备哪些能力。
@Replayable @RouteRegistry @VfResolvable @CustomState
↗
🛡️
AI 规范体系
34 条禁止项 + 分层规则体系，由 AI 在编码过程中实时约束。改 UI 时自动加载 UI 规范，改网络时自动加载网络规范。提交前自动化检查不可绕过。
禁止项拦截 按需加载规则 OpenSpec 流程
↗
🎨
自定义 UI + 极少重组
移除 Material3 组件库，自建轻量组件体系。智能跳过、延迟读取、计算缓存三策略联动，让只有数据真正变化的组件才刷新，其余全部跳过。
去 Material3 智能跳过 延迟读取 计算缓存

androidApp / iosApp
desktopApp		 平台入口（薄壳）
macrobenchmark Baseline Profile 生成 + 启动基准测试（Android）
tools/btrace btrace (RheaTrace) 3.0 性能追踪工具
tools/verify VF 截图回归：SSIM 对比引擎 + 录制 / 验证脚本
