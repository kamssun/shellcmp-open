Core Capabilities

Eight Core Systems

Each system works independently; together they form a complete engineering quality flywheel

↗
⏪
MVI State Restore — The Foundation
Strict unidirectional data flow architecture naturally produces complete operation history. Every user action is recorded, state changes fully traceable and replayable. The foundation for VF verification, recording system, server data固化—without this cornerstone, nothing else is possible.
Interaction Restore State Export/Import Dev Bookmarks Leak Audit Bug Replay
↗
🤖
AI Fully Automated Verification (VF)
Not just UI screenshot comparison—any bug causing UI changes gets caught. Records complete user operation path + server data snapshot, replays identically during verification. 10x faster than traditional E2E.
Change Analysis Path Inference Auto Recording SSIM Comparison Server Data固化
↗
⏱️
OBO Dispatcher — No Jank
Queue always has at most one task. When VSYNC arrives the queue is nearly empty, barrier mechanism works normally. Each task becomes independent message, performance monitoring can pinpoint "this task took 50ms".
Backlog Elimination 3rd SDK Intercept Source Diagnosis Three Platforms
↗
📊
Full-Chain Performance Diagnosis
Homegrown runtime performance monitoring, no third-party dependency, covers three platforms. Startup timeline tree shows each step's cost; each frame auto-decomposed into 6 stages aligned with btrace flame graphs; btrace + AI auto-analyzes trace data.
Startup Timeline Jank Diagnosis GC Pressure Frame Decomposition btrace + AI Slow Task Monitor
↗
📍
Fully Auto Tracking
Four event types (page, interaction, exposure, foreground/background) fully automated. New tracking list page = ExposureLazyColumn + appClickable, zero tracking code needed.
KSP Auto Mapping Three-layer Desensitization Exposure Tracking
↗
⚙️
KSP Code Generation
Four annotations eliminate massive duplicate code. New page with restore + verification = write business code + add two annotations, everything else auto-generated. Juniors see annotations and know this module's capabilities.
@Replayable @RouteRegistry @VfResolvable @CustomState
↗
🛡️
AI Spec System
34 forbidden items + layered rule system, enforced by AI during coding—not review after, but prevention during. Pre-commit checks are un绕过able.
Forbidden Intercept On-demand Rules OpenSpec Process
↗
🎨
Custom UI + Minimal Recomposition
Removed Material3, built lightweight component system. Smart skip, deferred read, computed cache three strategies combined—only components with actual data changes refresh, others all skip.
No Material3 Smart Skip Deferred Read Computed Cache
